# 退款与退票规则

文档类型：退款规则  
适用对象：乘客、管理员  
关键词：退款、退票、退改、退款资格、退款状态、审核流程

## 1. 退款申请基础规则

退款申请通常仅面向已支付订单开放。  
未支付订单不需要退款，通常可直接取消。  
已取消订单不能再次申请退款。

## 2. 可以申请退款的常见场景

以下场景通常支持申请退款：

- 已支付且尚未出行
- 已支付且待核销
- 已支付且业务规则允许已完成后申请售后退款

具体以平台退款策略和线路规则为准。

## 3. 不支持退款的常见场景

以下场景通常不支持退款：

- 未支付订单
- 已取消订单
- 已退款订单
- 同一订单已存在未处理退款申请
- 不满足平台特定时间窗口要求

## 4. 退款审核流程

退款流程通常包括：

1. 用户提交退款申请
2. 订单进入退款申请中状态
3. 管理员审核通过或驳回
4. 系统更新退款状态并通知用户

## 5. 退款状态说明

退款状态通常包括：

- `none`：无退款
- `requested`：退款申请中
- `refunded`：已退款
- `rejected`：退款被驳回

AI 在回答时应尽量使用面向用户的自然语言，如“退款申请中”“已退款”“退款被驳回”。

## 6. 驳回后的处理建议

若退款被驳回，用户可：

- 查看审核备注
- 补充材料后重新申请
- 联系平台客服进一步确认

若系统未开放重新申请入口，AI 应明确说明当前版本的处理方式。
