# TripVerse 前端接口文档

本文档用于前端和后端联调，重点说明前端页面会调用哪些接口，以及每个接口需要传入什么参数。

## 1. 用户中心模块

### 1.1 发送短信验证码

- 方法：`POST`
- 路径：`/api/auth/sms/send`

请求体：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `phone` | string | 是 | 手机号 |
| `scene` | string | 是 | 使用场景：`login` / `register` / `reset_password` |

### 1.2 注册

- 方法：`POST`
- 路径：`/api/auth/register`

请求体：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `role` | string | 是 | 角色：`passenger` / `driver` |
| `phone` | string | 是 | 手机号 |
| `smsCode` | string | 是 | 短信验证码 |
| `password` | string | 是 | 登录密码 |
| `nickname` | string | 否 | 用户昵称 |

### 1.3 密码登录

- 方法：`POST`
- 路径：`/api/auth/login/password`

请求体：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `role` | string | 否 | 当前登录角色，前端可传 `passenger` / `driver` |
| `phone` | string | 是 | 手机号 |
| `password` | string | 是 | 登录密码 |

### 1.4 验证码登录

- 方法：`POST`
- 路径：`/api/auth/login/code`

请求体：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `role` | string | 否 | 当前登录角色 |
| `phone` | string | 是 | 手机号 |
| `smsCode` | string | 是 | 短信验证码 |

### 1.5 获取当前登录用户

- 方法：`GET`
- 路径：`/api/auth/me`

请求头：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `Authorization` | string | 是 | Bearer Token |

### 1.6 退出登录

- 方法：`POST`
- 路径：`/api/auth/logout`

请求头：

| 参数 | 类型 | 必填 | 说明 |

### 1.7 获取用户资料

## 2. 票务查询模块

### 2.1 首页数据

- 方法：`GET`
- 路径：`/api/home`

查询参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `city` | string | 否 | 当前城市 |

### 2.2 票务搜索

- 方法：`GET`
- 路径：`/api/tickets/search`

查询参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `startCity` | string | 是 | 起点 |
| `endCity` | string | 是 | 终点 |
| `date` | string | 是 | 出发日期，格式 `YYYY-MM-DD` |
| `vehicleType` | string | 否 | 出行类型：汽车/高铁/火车/飞机/公交 |
| `sortBy` | string | 否 | 排序字段：`price` / `departureTime` / `duration` |
| `sortOrder` | string | 否 | 排序方向：`asc` / `desc` |
| `directOnly` | boolean | 否 | 是否只看直达 |
| `transferOnly` | boolean | 否 | 是否只看换乘 |
| `page` | number | 否 | 页码 |
| `pageSize` | number | 否 | 每页数量 |

### 2.3 班次详情

- 方法：`GET`
- 路径：`/api/tickets/{ticketId}`

路径参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `ticketId` | string | 是 | 班次 ID |

## 3. 订单模块

### 3.1 下单预览

- 方法：`POST`
- 路径：`/api/orders/preview`

请求体：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `ticketId` | string | 是 | 班次 ID |
| `seatType` | string | 是 | 座位类型 |
| `ticketCount` | number | 是 | 购票数量 |
| `passengers` | array | 是 | 乘车人列表 |
| `contactPhone` | string | 是 | 联系手机号 |
| `remark` | string | 否 | 备注 |

`passengers` 子项：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `name` | string | 是 | 乘车人姓名 |
| `idCard` | string | 是 | 身份证号 |
| `phone` | string | 否 | 乘车人手机号 |

### 3.2 创建订单

- 方法：`POST`
- 路径：`/api/orders`

请求体与预览一致，后端创建正式订单。

### 3.3 我的订单列表

- 方法：`GET`
- 路径：`/api/orders/my`

查询参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `status` | string | 否 | 订单状态 |
| `page` | number | 否 | 页码 |
| `pageSize` | number | 否 | 每页条数 |

### 3.4 订单详情

- 方法：`GET`
- 路径：`/api/orders/{orderId}`

路径参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `orderId` | string | 是 | 订单 ID |

### 3.5 取消订单

- 方法：`POST`
- 路径：`/api/orders/{orderId}/cancel`

路径参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `orderId` | string | 是 | 订单 ID |

请求体：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `reason` | string | 否 | 取消原因 |

### 3.6 申请退款

- 方法：`POST`
- 路径：`/api/orders/{orderId}/refund`

请求体：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `reason` | string | 是 | 退款原因 |
| `refundAmount` | number | 否 | 退款金额，前端可不传，由后端计算 |

### 3.7 改签

- 方法：`POST`
- 路径：`/api/orders/{orderId}/reschedule`

请求体：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `targetTicketId` | string | 是 | 目标班次 ID |
| `seatType` | string | 否 | 新座位类型 |
| `remark` | string | 否 | 改签备注 |
